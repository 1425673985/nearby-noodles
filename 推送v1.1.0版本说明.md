### 如果遇到SSL证书问题

可以临时禁用SSL验证（仅用于推送，不推荐长期使用）：

```bash
# 临时禁用SSL验证
git config --global http.sslVerify false

# 推送
git push
git push origin v1.1.0

# 推送完成后，恢复SSL验证
git config --global http.sslVerify true
```

### 或者使用SSH方式（推荐）

如果已配置SSH密钥，可以修改远程地址：

```bash
# 查看当前远程地址
git remote -v

# 如果使用HTTPS，可以改为SSH
git remote set-url origin git@github.com:1425673985/nearby-noodles.git

# 然后推送
git push
git push origin v1.1.0
```
