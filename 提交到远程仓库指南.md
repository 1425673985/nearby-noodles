## 常见问题

### 1. 如果远程仓库已存在文件

如果远程仓库已经初始化了README等文件，需要先拉取：

```bash
git pull origin main --allow-unrelated-histories
# 解决可能的冲突后
git push -u origin main
```

### 2. 认证问题

- **HTTPS方式**：会提示输入用户名和密码（GitHub需要使用Personal Access Token）
- **SSH方式**：需要先配置SSH密钥

### 3. 分支名称问题

如果远程默认分支是`master`而不是`main`：

```bash
# 方法1：重命名本地分支
git branch -M master
git push -u origin master

# 方法2：推送到master分支
git push -u origin main:master
```

## 注意事项

1. **敏感信息**：确保`.gitignore`已配置，不要提交：
   - API Key
   - 密码
   - 个人配置

2. **提交信息**：使用有意义的提交信息，方便后续查看历史

3. **定期推送**：建议每次完成功能后及时推送，避免本地丢失

4. **分支管理**：可以创建功能分支开发，完成后合并到main分支
